Topic 1 Introduction to Database

▼
📖 Database (DB) A Database is an organised collection of related data stored in a structured format so that it can be easily accessed, managed, and updated. It eliminates redundancy and ensures data integrity.
📖 Database Management System (DBMS) A DBMS is a software system that enables users to define, create, maintain, and control access to a database. Examples: MySQL, Oracle, MS Access, PostgreSQL, SQLite.
⭐ Why DBMS? — Advantages over File System
  • Reduced Data Redundancy — Same data not stored multiple times.
  • Data Consistency — Change once, reflected everywhere.
  • Data Integrity — Rules ensure only valid data is stored.
  • Data Security — Access control prevents unauthorised use.
  • Data Sharing — Multiple users can access simultaneously.
  • Backup & Recovery — Automatic backup mechanisms.
  • Data Independence — App programs independent of data storage.

Disadvantages of Traditional File System

  • Data Redundancy and Inconsistency
  • Difficulty in accessing data
  • Data isolation — data scattered in multiple files
  • Integrity problems
  • Atomicity problems
  • Concurrent access anomalies
  • Security problems

Topic 2 Relational Database Management System (RDBMS)

▼
📖 RDBMS An RDBMS is a type of DBMS that stores data in the form of tables (relations). Tables are linked using keys. It is based on E.F. Codd's relational model.

Popular RDBMS: MySQL, Oracle, SQL Server, PostgreSQL, MariaDB, SQLite.

Topic 3 Keys in RDBMS

▼
⭐ Keys — Most Important for Exam!
Keys are used to uniquely identify rows in a table and to establish relationships between tables.
🔑 1. Candidate Key An attribute (or set of attributes) that can uniquely identify each row in a table. A table can have multiple candidate keys.
Properties: Unique + Not Null.
Example: In STUDENT table — RollNo and Email both can be candidate keys.
🔑 2. Primary Key (PK) The candidate key chosen by the designer to uniquely identify each row. A table has only one primary key.
Properties: Unique + Not Null.
Example: RollNo is chosen as Primary Key in STUDENT table.
🔑 3. Alternate Key All candidate keys except the primary key are called Alternate Keys.
Example: If RollNo is PK, then Email becomes the Alternate Key.
🔑 4. Foreign Key (FK) An attribute in one table that is the Primary Key of another table. It creates a link between two tables.
Example: In MARKS table, RollNo refers to RollNo in STUDENT table — so RollNo in MARKS is a Foreign Key.
Referential Integrity: FK value must exist in the referenced table or be NULL.
🔑 5. Composite Key A key made of two or more attributes that together uniquely identify a row.
Example: (StudentID + SubjectCode) together can be a composite key in EXAM table.
🔑 6. Super Key Any set of attributes that can uniquely identify a row. Every Primary Key is a Super Key but vice-versa is not always true.
Example: {RollNo}, {RollNo, Name}, {RollNo, Age} — all are super keys.

Topic 4 Data Models & ER Diagram

▼

Types of Data Models

🗂️ Hierarchical Model

Data is organised in a tree structure (parent-child). One parent, multiple children. Example: File system.

  • Fast access
  • Rigid structure
  • Complex many-to-many relations
🔗 Network Model

Data is organised as a graph. Records can have multiple parent and child records. Overcomes hierarchical limitations.

  • Flexible
  • Complex to design
  • Hard to modify
📋 Relational Model

Data stored in tables (relations). Tables linked via keys. Based on E.F. Codd's 12 rules. Most popular today.

  • Simple tabular format
  • Easy to query (SQL)
  • MySQL, Oracle use this
🔷 Object-Oriented Model

Data stored as objects like in OOP. Objects have attributes and methods. Used in complex applications.

  • Handles complex data
  • Reusability
  • Less common for CBSE

⭐ CHAR vs VARCHAR — Important Difference
  • CHAR(n): Always stores exactly n characters. If input is shorter, pads with spaces. Faster but wastes space.
  • VARCHAR(n): Stores only the actual characters entered (up to n). Saves space. Slightly slower.
  • Example: CHAR(10) for "Ram" stores "Ram " (7 spaces added). VARCHAR(10) stores "Ram" only.

⭐ Importance of Communication
Communication skills are needed to:

Perspectives in Communication

📖 Perspectives Perspectives are ideas, views, or fixed ways of thinking that sometimes affect our communication. For example, if your teacher is usually strict, even when they are being friendly, you may think they are scolding you.
⭐ Factors Affecting Perspectives in Communication
  • Language — Incorrect words, unfamiliar language, or lack of detail can act as a barrier.
  • Visual Perception — The brain's ability to make sense of what we see. E.g., partially drawn pictures can be misunderstood.
  • Past Experience — Earlier experiences can stop us from understanding clearly. E.g., "This shopkeeper cheated me last time."
  • Prejudice — Fixed ideas like "No one in my class likes me" may stop a student from communicating openly.
  • Feelings — Emotions like lack of interest or distrust affect communication. E.g., "I am not feeling well."
  • Environment — Noise or disturbance in surroundings can make communication difficult.
  • Personal Factors — Your own feelings, habits, and ways of thinking.
  • Culture — Signs mean different things in different cultures. E.g., showing a thumb means 'good job' for some but may be insulting to others.

Session 2 Climate Change & Global Warming

▼
📖 What is Climate Change? Climate change refers to long-term shifts in global temperatures and weather patterns. While some changes are natural, human activities since the 1800s have been the main cause.
📖 Greenhouse Effect & Global Warming The Greenhouse Effect is the trapping of the sun's heat in the Earth's atmosphere by gases like CO₂ and methane — causing the planet to warm up. Excessive warming is called Global Warming.
⭐ Causes of Climate Change
  • Burning of fossil fuels (coal, oil, petrol) — releases CO₂
  • Deforestation — reduces CO₂ absorption by trees
  • Industrial activities — release harmful gases
  • Agriculture — livestock and farming release methane
  • Vehicles — release carbon emissions
⚠️ Effects of Climate Change
  • Rising sea levels due to melting glaciers
  • Extreme weather events (floods, droughts, cyclones)
  • Loss of biodiversity and extinction of species
  • Crop failures and food insecurity
  • Spread of diseases

Topic 3 Number System Conversions — Decimal ↔ Binary / Octal / Hex

▼
📌 Method: Decimal → Any Base (Division Method)
  1. Divide the number by the base (2 / 8 / 16).
  2. Note the remainder.
  3. Keep dividing quotient by base until it becomes 0; note each remainder.
  4. Write all remainders in reverse order (bottom to top) — that is the answer.
🔄 Decimal → Binary (÷ 2)
Example 1: (45)₁₀ → Binary
45 ÷ 2 → remainders read bottom to top = (101101)₂
Example 2: (234)₁₀ → Binary

📌 Method: Any Base → Decimal (Positional Value Method)
  1. Write down the number.
  2. Starting from right to left, assign position numbers (0, 1, 2, 3…).
  3. Positional value = digit × baseposition.
  4. Add all positional values to get the decimal equivalent.
🔄 Binary → Decimal

Example 2: (111100)₂ → Decimal

🟣 Cache Memory A very high-speed type of primary memory placed between CPU and RAM. Its main purpose is to store copies of frequently accessed data from RAM, reducing the time taken to access that data by the CPU.
⭐ Cache Memory — Key Points
  • Faster than RAM but smaller in size.
  • Located between CPU and Primary Memory (RAM).
  • Stores frequently used data/instructions for quick access.
  • Reduces CPU wait time — improves overall system speed.
  • Types: L1 Cache (inside CPU core), L2 Cache, L3 Cache.
🟠 Secondary Memory (Storage) Stores data and programs permanently. It is non-volatile. CPU cannot access secondary memory directly — data must first be loaded into primary memory.
⭐ Secondary Memory — Key Points
  • Non-volatile — data retained even after power off.
  • Larger storage capacity than primary memory.
  • Slower access speed compared to primary memory.
  • CPU cannot directly access it; data must be copied to RAM first.
  • Examples: SSD (Solid State Drive), HDD (Hard Disk Drive), CD/DVD, Pen Drive/Flash Drive, Memory Card.
