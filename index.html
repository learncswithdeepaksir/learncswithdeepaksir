Topic 1 Introduction to Database

▼
📖 Database (DB) A Database is an organised collection of related data stored in a structured format so that it can be easily accessed, managed, and updated. It eliminates redundancy and ensures data integrity.
📖 Database Management System (DBMS) A DBMS is a software system that enables users to define, create, maintain, and control access to a database. Examples: MySQL, Oracle, MS Access, PostgreSQL, SQLite.
⭐ Why DBMS? — Advantages over File System
  • Reduced Data Redundancy — Same data not stored multiple times.
  • Data Consistency — Change once, reflected everywhere.
  • Data Integrity — Rules ensure only valid data is stored.
  • Data Security — Access control prevents unauthorised use.
  • Data Sharing — Multiple users can access simultaneously.
  • Backup & Recovery — Automatic backup mechanisms.
  • Data Independence — App programs independent of data storage.

Disadvantages of Traditional File System

  • Data Redundancy and Inconsistency
  • Difficulty in accessing data
  • Data isolation — data scattered in multiple files
  • Integrity problems
  • Atomicity problems
  • Concurrent access anomalies
  • Security problems

Topic 2 Relational Database Management System (RDBMS)

▼
📖 RDBMS An RDBMS is a type of DBMS that stores data in the form of tables (relations). Tables are linked using keys. It is based on E.F. Codd's relational model.

Popular RDBMS: MySQL, Oracle, SQL Server, PostgreSQL, MariaDB, SQLite.

Topic 3 Keys in RDBMS

▼
⭐ Keys — Most Important for Exam!
Keys are used to uniquely identify rows in a table and to establish relationships between tables.
🔑 1. Candidate Key An attribute (or set of attributes) that can uniquely identify each row in a table. A table can have multiple candidate keys.
Properties: Unique + Not Null.
Example: In STUDENT table — RollNo and Email both can be candidate keys.
🔑 2. Primary Key (PK) The candidate key chosen by the designer to uniquely identify each row. A table has only one primary key.
Properties: Unique + Not Null.
Example: RollNo is chosen as Primary Key in STUDENT table.
🔑 3. Alternate Key All candidate keys except the primary key are called Alternate Keys.
Example: If RollNo is PK, then Email becomes the Alternate Key.
🔑 4. Foreign Key (FK) An attribute in one table that is the Primary Key of another table. It creates a link between two tables.
Example: In MARKS table, RollNo refers to RollNo in STUDENT table — so RollNo in MARKS is a Foreign Key.
Referential Integrity: FK value must exist in the referenced table or be NULL.
🔑 5. Composite Key A key made of two or more attributes that together uniquely identify a row.
Example: (StudentID + SubjectCode) together can be a composite key in EXAM table.
🔑 6. Super Key Any set of attributes that can uniquely identify a row. Every Primary Key is a Super Key but vice-versa is not always true.
Example: {RollNo}, {RollNo, Name}, {RollNo, Age} — all are super keys.

Topic 4 Data Models & ER Diagram

▼

Types of Data Models

🗂️ Hierarchical Model

Data is organised in a tree structure (parent-child). One parent, multiple children. Example: File system.

  • Fast access
  • Rigid structure
  • Complex many-to-many relations
🔗 Network Model

Data is organised as a graph. Records can have multiple parent and child records. Overcomes hierarchical limitations.

  • Flexible
  • Complex to design
  • Hard to modify
📋 Relational Model

Data stored in tables (relations). Tables linked via keys. Based on E.F. Codd's 12 rules. Most popular today.

  • Simple tabular format
  • Easy to query (SQL)
  • MySQL, Oracle use this
🔷 Object-Oriented Model

Data stored as objects like in OOP. Objects have attributes and methods. Used in complex applications.

  • Handles complex data
  • Reusability
  • Less common for CBSE

⭐ CHAR vs VARCHAR — Important Difference
  • CHAR(n): Always stores exactly n characters. If input is shorter, pads with spaces. Faster but wastes space.
  • VARCHAR(n): Stores only the actual characters entered (up to n). Saves space. Slightly slower.
  • Example: CHAR(10) for "Ram" stores "Ram " (7 spaces added). VARCHAR(10) stores "Ram" only.

Topic 7 DML Commands — INSERT, UPDATE, DELETE, SELECT

▼

INSERT Command

-- Insert all columns (order must match table structure) INSERT INTO STUDENT VALUES (101, 'Aman Sharma', 17, 'Delhi', 89.5, 'M', '2007-05-14'); -- Insert specific columns INSERT INTO STUDENT (RollNo, Name, Marks) VALUES (102, 'Priya', 92); -- Insert multiple rows INSERT INTO STUDENT VALUES (103, 'Rohan', 17, 'Delhi', 78, 'M', '2007-01-20'), (104, 'Neha', 16, 'Lucknow', 95, 'F', '2008-03-11');

SELECT Command (Most Important)

-- Select all columns SELECT * FROM STUDENT; -- Select specific columns SELECT Name, Marks FROM STUDENT; -- WHERE clause — filter rows SELECT * FROM STUDENT WHERE City = 'Delhi'; SELECT * FROM STUDENT WHERE Marks > 80; SELECT * FROM STUDENT WHERE Age BETWEEN 15 AND 18; SELECT * FROM STUDENT WHERE Name LIKE 'A%'; -- starts with A SELECT * FROM STUDENT WHERE Name LIKE '%a'; -- ends with a SELECT * FROM STUDENT WHERE Name LIKE '_o%'; -- 2nd char is o SELECT * FROM STUDENT WHERE City IN ('Delhi', 'Mumbai'); SELECT * FROM STUDENT WHERE Age IS NULL; -- ORDER BY — sort results SELECT * FROM STUDENT ORDER BY Marks DESC; -- highest first SELECT * FROM STUDENT ORDER BY Name ASC; -- alphabetical -- DISTINCT — remove duplicates SELECT DISTINCT City FROM STUDENT; -- LIMIT — retrieve limited rows SELECT * FROM STUDENT LIMIT 3;

Aggregate Functions with GROUP BY

FunctionPurposeExample
COUNT()Count number of rowsSELECT COUNT(*) FROM STUDENT;
SUM()Sum of values in a columnSELECT SUM(Marks) FROM STUDENT;
AVG()Average of valuesSELECT AVG(Marks) FROM STUDENT;
MAX()Maximum valueSELECT MAX(Marks) FROM STUDENT;
MIN()Minimum valueSELECT MIN(Age) FROM STUDENT;
-- GROUP BY — group rows by a column SELECT City, COUNT(*) AS Total FROM STUDENT GROUP BY City; SELECT City, AVG(Marks) FROM STUDENT GROUP BY City; -- HAVING — filter after GROUP BY (like WHERE for groups) SELECT City, COUNT(*) FROM STUDENT GROUP BY City HAVING COUNT(*) > 1;
⭐ WHERE vs HAVING — Important!
  • WHERE filters rows before grouping. Cannot use aggregate functions.
  • HAVING filters groups after GROUP BY. Can use aggregate functions.
  • Example: WHERE Marks > 50 → filters individual rows. HAVING AVG(Marks) > 50 → filters groups.

UPDATE Command

-- Update specific rows UPDATE STUDENT SET Marks = 95 WHERE RollNo = 101; -- Update multiple columns UPDATE STUDENT SET City = 'Chennai', Age = 18 WHERE RollNo = 102; -- Update all rows (no WHERE — be careful!) UPDATE STUDENT SET Marks = Marks + 5;

DELETE Command

-- Delete specific rows DELETE FROM STUDENT WHERE RollNo = 103; -- Delete rows matching condition DELETE FROM STUDENT WHERE Marks < 33; -- Delete all rows (keeps table structure) DELETE FROM STUDENT;

Topic 8 SQL Joins & Normalization

▼

SQL Joins

📖 JOIN A JOIN is used to retrieve data from two or more tables based on a related column (usually a Foreign Key – Primary Key pair).
Join TypeReturns
INNER JOIN (JOIN)Only matching rows from BOTH tables
LEFT JOINAll rows from left table + matching from right (NULL if no match)
RIGHT JOINAll rows from right table + matching from left (NULL if no match)
CROSS JOINCartesian product — every row of left × every row of right
-- INNER JOIN — common rows only SELECT S.Name, S.City, M.Subject, M.Score FROM STUDENT S INNER JOIN MARKS M ON S.RollNo = M.RollNo; -- LEFT JOIN — all students, even without marks SELECT S.Name, M.Score FROM STUDENT S LEFT JOIN MARKS M ON S.RollNo = M.RollNo; -- Cartesian Product (old syntax) SELECT * FROM STUDENT, MARKS;

Normalization

📖 Normalization The process of organising a database to reduce data redundancy and improve data integrity by dividing large tables into smaller related tables.
Normal FormCondition
1NF (First Normal Form)Each column must have atomic (indivisible) values. No repeating groups.
2NF (Second Normal Form)Must be in 1NF + no partial dependency (every non-key attribute fully depends on the whole primary key).
3NF (Third Normal Form)Must be in 2NF + no transitive dependency (non-key attributes should not depend on other non-key attributes).

Important MySQL Commands for Reference

-- Show all databases SHOW DATABASES; -- Show all tables in current database SHOW TABLES; -- See structure/schema of a table DESCRIBE STUDENT; DESC STUDENT; -- See the CREATE statement used for a table SHOW CREATE TABLE STUDENT; -- Column alias (rename in output) SELECT Name AS StudentName, Marks AS Score FROM STUDENT; -- String functions SELECT UPPER(Name), LOWER(City), LENGTH(Name) FROM STUDENT; SELECT SUBSTRING(Name, 1, 3) FROM STUDENT; SELECT CONCAT(Name, ' - ', City) FROM STUDENT; -- Math / Numeric functions SELECT ROUND(89.67, 1); -- 89.7 SELECT MOD(10, 3); -- 1 SELECT POWER(2, 8); -- 256 -- Date functions SELECT NOW(); -- current date & time SELECT CURDATE(); -- today's date SELECT YEAR(DOB) FROM STUDENT; -- extract year from date

Transaction Control Language (TCL)

📖 Transaction A Transaction is a sequence of one or more SQL statements that are executed as a single unit. Either all succeed (COMMIT) or all fail (ROLLBACK) — this is called Atomicity.
START TRANSACTION; -- Begin transaction UPDATE STUDENT SET Marks = 90 WHERE RollNo = 101; DELETE FROM STUDENT WHERE RollNo = 999; SAVEPOINT sp1; -- Create a save point ROLLBACK TO sp1; -- Undo changes after sp1 ROLLBACK; -- Undo ALL changes in transaction COMMIT; -- Save all changes permanently
📌 ACID Properties — Important for Exam!
PropertyMeaning
A — AtomicityAll operations in a transaction complete or none do (all-or-nothing).
C — ConsistencyDatabase moves from one valid state to another; constraints always met.
I — IsolationConcurrent transactions do not interfere with each other.
D — DurabilityOnce committed, changes are permanent even after system failure.

Session 1 Introduction to Communication

▼
📖 What is Communication? The word 'Communication' comes from the Latin word commūnicāre, meaning 'to share'. Communication is the exchange of information through signs, signals, speech, writing, or other forms of media.
⭐ Importance of Communication
Communication skills are needed to:
  • Inform — Give facts or information to someone.
  • Influence — Change or persuade someone in an indirect but important way.
  • Express Feelings — Talking about your feelings is a healthy way to express them.

Elements of Communication

Communication is a two-way exchange of information — giving and receiving.

ElementDescription
SenderThe person who starts the communication process by sending a message.
MessageThe information, idea, or feeling the sender wants to convey.
ChannelThe path/medium used (phone, face-to-face, writing, email, etc.).
ReceiverThe person who receives and understands the message.

Perspectives in Communication

📖 Perspectives Perspectives are ideas, views, or fixed ways of thinking that sometimes affect our communication. For example, if your teacher is usually strict, even when they are being friendly, you may think they are scolding you.
⭐ Factors Affecting Perspectives in Communication
  • Language — Incorrect words, unfamiliar language, or lack of detail can act as a barrier.
  • Visual Perception — The brain's ability to make sense of what we see. E.g., partially drawn pictures can be misunderstood.
  • Past Experience — Earlier experiences can stop us from understanding clearly. E.g., "This shopkeeper cheated me last time."
  • Prejudice — Fixed ideas like "No one in my class likes me" may stop a student from communicating openly.
  • Feelings — Emotions like lack of interest or distrust affect communication. E.g., "I am not feeling well."
  • Environment — Noise or disturbance in surroundings can make communication difficult.
  • Personal Factors — Your own feelings, habits, and ways of thinking.
  • Culture — Signs mean different things in different cultures. E.g., showing a thumb means 'good job' for some but may be insulting to others.

Effective communication can happen if we follow the 7C principles of professional communication:

This shows non-verbal communication is more powerful than verbal!

Session 6 Pronunciation Basics

📖 Pronunciation Pronunciation is the way we speak a word — the sounds we make and the stress we place on certain syllables. Correct pronunciation helps others understand us clearly.
📖 Phonetics Phonetics is the study of the sounds of speech. It helps us understand how to correctly pronounce words. Each sound in a language is represented by a phonetic symbol.
⭐ Key Pronunciation Tips
  • Speak clearly and at a moderate speed — not too fast, not too slow.
  • Pay attention to stress in words. E.g., PREsent (noun) vs preSENT (verb).
  • Words can have silent letters. E.g., the 'k' in 'knife' is silent.
  • Practice with a dictionary that shows phonetic symbols.
  • Listen carefully to how native or fluent speakers pronounce words.
